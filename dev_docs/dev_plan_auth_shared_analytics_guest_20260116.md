# Read-Tube 全面升级：UI 驱动实施计划 (最终修正版)

本计划强调 **“视觉原型先行，数据逻辑后补”**。通过高保真 UI 与模拟数据，在正式建表前锁定业务指标与交互深度。

## 升级版 UI 设计阶段 (P0 - 当前重点)

### 1. 登录页 (Gateway)
*   **元素**：Google 登录、邮件表单、以及显眼的 **“Guest 体验”按钮**。
*   **目标**：明确多种登录入口及访客接入流程。

### 2. 统一仪表盘 (Unified Dashboard)
*   **设计原则**：Guest 与普通用户共用同一套 UI 模版。
*   **全局导航逻辑**：在任何页面点击 Logo 或图标，系统需智能判断跳转目标：
    *   **已登录用户**：跳转至个人仪表盘 (Logged-in Home)。
    *   **未登录/游客**：跳转至公共营销首页。
*   **显示模式切换**：用户可一键切换“网格视图（缩略图+标题）”与“列表视图（仅标题）”，系统需记录用户的视觉偏好。
*   **逻辑**：如果是 Guest，自动填充库中最近 5 条公开 YouTube 报告。
*   **元素**：视频管理卡片、权限切换开关 (Share Toggle)、统计概览。

### 3. 管理者汇总页 (Admin Insight)
*   **定位**：独立的统计看板，用于全站热度与行为分析。
*   **图表设计**：先使用 **模拟 JSON 数据** 填充，展示全网点击排行榜、段落点击热力图等。
*   **时序**：排在仪表盘设计之后，以便根据仪表盘的交互深度来决定统计维度。

### 4. 增强版报告页
*   **重心**：讨论区 UI、互动按钮、以及埋点捕获逻辑。

## 实施路径与节奏

> [!IMPORTANT]
> **代码驱动预览 (Code-First Preview)**：
> 1. **高保真代码原型**：在前端 (Next.js) 中直接编写真实的 UI 代码，使用 Mock 数据模拟业务逻辑。
> 2. **真机预览与微调**：在本地运行 `npm run dev` 观察真实网页效果。
> 3. **数据模型对齐**：在 UI 代码完全定稿后，根据组件的需求反向推导并执行官方数据库 SQL。

## 详细路径

### 第一阶段：前端代码原型开发 (P0 - 当前重点)
*   实现 **登录页**：包含 Google/邮箱/Guest 登录交互。
*   实现 **统一仪表盘**：使用 Mock 数据展示视频列表，包含布局切换逻辑。
*   实现 **管理者看板**：使用模拟图表展示热度指标。
*   实现 **增强报告页**：完成讨论区与视频互动的前端样式。

### 第二阶段：数据库建模 (P1)
*   根据 UI 设计定稿 SQL 脚本并执行。

### 第三阶段：功能落地 (P2)
*   Supabase 集成、数据对接、评论系统实现。

### 第四阶段：埋点与高级分析 (P3)
*   离线埋点系统与正式数据统计。
