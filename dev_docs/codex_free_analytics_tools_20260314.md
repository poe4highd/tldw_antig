# 免费网站访问分析工具梳理（Codex，2026-03-14）

## 背景

你这次要的是“先讨论，不接代码”，目标是梳理有哪些**免费**工具可以帮助站点收集访问数据并做分析，同时结合当前项目的语境判断哪些更值得先上。

结合现有 [SEO 英文市场战略](./seo-english-market-strategy.md)，这个项目当前更像是一个：

- 依赖 Google 搜索获取新流量的内容型/工具型网站
- 需要同时看 SEO 表现、站内行为、转化路径
- 未来可能会关心隐私合规与脚本体积

所以这里不只看“能不能统计 PV/UV”，而是把工具分成几类来看：

- 搜索表现与索引
- 通用流量分析
- 行为分析与 UX 诊断
- 隐私友好轻量统计
- 自托管开源方案

---

## 先给结论

如果只想快速得到一套够用、且基本不花钱的组合，我的判断是：

1. **必装：Google Search Console**
2. **主分析：Google Analytics 4（GA4）**
3. **行为诊断：Microsoft Clarity**
4. **可选补充：Cloudflare Web Analytics**

如果你更在意**隐私、数据自主可控、减少对 Google 依赖**，则可以考虑：

1. **Google Search Console**
2. **Umami（自托管）或 Matomo On-Premise**
3. **是否再加 Clarity，取决于你要不要看录屏/热图**

原因很简单：

- `GSC` 解决“Google 怎么看你”的问题
- `GA4` 解决“用户从哪来、做了什么”的问题
- `Clarity` 解决“用户到底卡在哪”的问题
- `Cloudflare Web Analytics` 解决“轻量、隐私友好、低维护”的问题
- `Umami / Matomo` 解决“数据在自己手里”的问题

---

## 工具逐个看

### 1. Google Analytics 4（GA4）

**定位**：最常见的免费通用网站分析工具。

**适合看什么**

- 流量来源：自然搜索、直接访问、外链、社媒、广告
- 用户行为：页面浏览、停留、事件、转化
- 实时访问
- 漏斗、归因、受众分群这类偏增长问题

**优点**

- 免费版对多数中小站已经够用
- 生态最完整，和 Google Ads / Search / Tag 等体系衔接强
- 事件模型比老版 UA 更适合现代站点
- 如果以后你要认真做转化分析，GA4 还是主流基建

**缺点**

- 学习成本明显高，UI 和指标定义对新手不友好
- 很多人第一次上手会觉得“数据很多，但不好读”
- 对内容站来说，SEO 诊断能力不如 Search Console 直接
- 隐私和 Cookie 合规压力通常比轻量方案更高

**对你这个项目的价值**

- 适合看 `首页 -> 提交 -> 结果页 -> 注册/回访` 这类路径
- 适合对比不同渠道带来的用户质量
- 如果以后会做 landing page、A/B、转化优化，它很有用

**我的判断**

- 如果你接受 Google 生态，`GA4` 依然值得装
- 但它**不能替代 Search Console**

---

### 2. Google Search Console（GSC）

**定位**：SEO 必装，不是通用埋点工具，而是 Google 搜索侧的官方观察台。

**适合看什么**

- 哪些关键词把用户带进来
- 每个页面的点击、展现、CTR、平均排名
- 页面是否被索引
- sitemap 提交情况
- Core Web Vitals、页面体验、索引问题

**优点**

- 免费
- 官方搜索数据，SEO 视角最直接
- 对内容页、结果页、长尾词机会判断非常关键
- 能帮你验证 sitemap、抓取、索引与排名变化

**缺点**

- 只能看 Google 搜索相关表现
- 不能替代站内行为分析
- 数据粒度和回溯方式有自己的限制

**对你这个项目的价值**

- 当前项目明显应该优先关心：
  - 哪些 `result` 页面开始获得曝光
  - 哪些查询词把流量带进来
  - 哪些页面有展现但 CTR 低
  - 哪些页面未索引或索引不稳定

**我的判断**

- 这是你最应该先启用的免费工具，没有之一
- 如果只装一个，先装 `GSC`

---

### 3. Microsoft Clarity

**定位**：免费行为分析工具，强项是**录屏、热图、摩擦点定位**。

**适合看什么**

- 用户在哪些按钮、区域、表单上卡住
- 页面滚动深度
- 点击热图、滚动热图
- 会话录屏
- Rage click、快速退出、异常交互

**优点**

- 官方首页明确写了 `Free forever`
- 热图和录屏对产品改版非常直观
- 很适合排查“为什么用户没继续提交/没点注册/没看完结果页”
- 和 GA4 不是替代关系，而是互补关系

**缺点**

- 不是 SEO 工具
- 不是深度归因工具
- 录屏类工具天然会带来更高的隐私敏感度，需要谨慎处理

**对你这个项目的价值**

- 你现在这种“工具站 + 内容页”模式，Clarity 特别适合看：
  - 首页 CTA 是否足够清晰
  - 用户是否在提交入口前流失
  - 结果页哪里最吸引注意力
  - Transcript / Summary / 时间轴 / 登录按钮是否被真正使用

**我的判断**

- 如果你要做产品体验优化，它的投入产出比很高
- 很适合作为 `GA4` 的补充

---

### 4. Cloudflare Web Analytics

**定位**：轻量、隐私友好、低维护的免费统计。

**适合看什么**

- 基础访问信息
- 页面表现
- 真实用户性能体验（RUM）

**优点**

- 官方文档明确写的是 `free, privacy-first analytics`
- 不要求把站点托管在 Cloudflare 代理后面
- 只需要 Cloudflare 账号和一段 JS
- 不收集或使用访客个人数据
- 非常适合作为“轻量补充层”

**缺点**

- 深度分析能力明显弱于 GA4
- 更适合看概况，不适合做复杂增长分析
- 事件、漏斗、归因这类能力不是它的强项

**对你这个项目的价值**

- 如果你想先低成本知道“页面有人看吗、性能怎么样、国家/来源大概如何”，它很好用
- 如果你对隐私和脚本体积比较敏感，它比 GA4 更轻

**我的判断**

- 它适合当“轻量统计层”
- 不适合单独承担完整增长分析任务

---

### 5. Umami（自托管）

**定位**：开源、隐私优先、可自托管的网站分析工具。

**适合看什么**

- 页面浏览、访客、来源、设备、国家
- 自定义事件
- 漏斗、用户旅程、留存、UTM

**优点**

- 官方文档明确强调：
  - 开源
  - 无 Cookie
  - 不采集个人数据
  - 可自托管
- 脚本轻
- 对开发者友好
- 比 GA4 简洁，学习成本低很多

**缺点**

- 你需要自己部署和维护
- 生态整合度不如 Google 体系
- 报表丰富度通常不如 GA4 全家桶

**对你这个项目的价值**

- 如果你想避免“又上 Google、又上第三方 SaaS”的重依赖，Umami 很适合
- 对一个 Next.js / Web 产品来说，接入成本通常不高

**我的判断**

- 它是很好的 `GA4` 替代候选
- 特别适合“想看流量，但不想把事情搞复杂”的团队

---

### 6. Matomo On-Premise

**定位**：更完整、更重型的自托管分析平台。

**适合看什么**

- 通用网站分析
- 更接近传统企业分析平台的报表诉求
- 强调数据所有权和隐私可控

**优点**

- 官方明确写了 `On-Premise` 版本可免费下载，并且会一直免费
- 数据自主可控能力强
- 功能面通常比轻量工具更全
- 对“我要一个不依赖 Google 的完整分析平台”这类需求很合适

**缺点**

- 比 Umami 更重，维护成本更高
- 需要服务器、数据库、升级和性能管理
- 如果你现在只是小团队/单人项目，可能会显得过度

**对你这个项目的价值**

- 如果后续你非常在意数据主权，或者需要更完整的分析后台，它是稳妥方案
- 但就目前项目阶段看，可能偏重

**我的判断**

- `Matomo` 更像“我要长期自建分析平台”
- `Umami` 更像“我要轻量自托管替代 GA4”

|---|---|---|---|---|
| Google Search Console | 免费 | SEO、索引、查询词、CTR、排名 | 不看站内行为 | 是，必须 |

## 针对这个项目，我会怎么选

### 方案 A：最务实的组合

- `Google Search Console`
- `Google Analytics 4`
- `Microsoft Clarity`

**适合情况**

- 你主要目标是尽快看 SEO 增长、站内行为和转化路径
- 接受使用 Google / Microsoft 的第三方分析服务

**为什么是这个组合**

- `GSC` 看搜索表现
- `GA4` 看渠道和站内行为
- `Clarity` 看真实交互问题

这三者的职责边界很清楚，不冲突。

---

### 方案 B：更偏隐私和轻量

- `Google Search Console`
- `Cloudflare Web Analytics`
- `Microsoft Clarity`

**适合情况**

- 你想先尽量少维护、少配置
- 暂时不想深入折腾 GA4 的事件体系

**问题**

- 后续如果要认真看漏斗、转化、用户分层，还是可能回到 GA4 或 Umami

---

### 方案 C：减少外部依赖

- `Google Search Console`
- `Umami` 或 `Matomo On-Premise`
- `是否加 Clarity，取决于你要不要录屏热图`

**适合情况**

- 你更重视数据控制权
- 可以接受自己部署与维护

**我的取舍**

- 当前阶段优先 `Umami`
- 未来如果分析诉求明显变复杂，再考虑 `Matomo`

---

## 一个现实判断：哪些是“必须”，哪些是“锦上添花”

### 必须

- `Google Search Console`

没有它，你几乎是在盲做 SEO。

### 高优先级

- `GA4` 或 `Umami`

你总得有一个“站内行为分析主系统”。

### 很值，但不是第一顺位

- `Clarity`

当你开始改首页、结果页、CTA、注册流程时，它会非常有价值。

### 可选补充

- `Cloudflare Web Analytics`

它更适合补充，不适合作为唯一核心分析系统。

---

## 合规和实际运营层面的提醒

这个部分不是法律意见，只是工程和产品上的现实提醒。

- `GA4`、`Clarity` 这类第三方分析工具，通常都需要你同步更新隐私政策说明。
- 如果你面向欧盟用户，Cookie / 同意管理会比“只看功能能不能用”更重要。
- `Clarity` 这类录屏工具在隐私敏感度上高于纯 PV/事件统计，部署时要关注是否需要屏蔽敏感字段。
- 如果你后面要做英文市场 SEO，`GSC` 的查询词、国家分布、索引状态会比单纯 PV 数据更有决策价值。

---

## 最后建议

如果现在只能排一个最小可行顺序，我建议是：

1. **先上 Google Search Console**
2. **再上 GA4**
3. **然后补 Clarity**
4. **如果你后面更在意隐私或想降低外部依赖，再评估 Umami**

这是因为你当前最直接的问题不是“高级产品分析”，而是：

- 页面有没有被 Google 看见
- 哪些页面开始拿到曝光
- 哪些关键词在带流量
- 用户进站后卡在什么地方

从这个角度看，`GSC + GA4 + Clarity` 是最现实的一组免费组合。
