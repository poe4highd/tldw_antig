# Audio Quick Reader 功能概览与使用指南

本文档总结了 **Audio Quick Reader** (v2.5+) 的核心功能细节与技术特点，旨在帮助用户和开发者快速了解系统的能力。

---

## 1. 核心功能特性

### 🎥 媒体处理与转录
*   **多源输入支持**：
    *   **YouTube 链接**：直接粘贴 URL，系统自动提取元数据（标题、封面）并下载处理。
    *   **本地上传**：支持视频（MP4, MOV, MKV）和音频（MP3, M4A, WAV）文件。
*   **智能视频缩略图**：
    *   对于上传的视频文件，后台自动调用 `ffmpeg` 截取第 1 秒处的图像作为预览封面。
*   **音轨自动分离**：
    *   上传视频时，系统会自动提取高质量音频用于 AI 转录，同时保留原始视频用于播放。
*   **多模式转录引擎**：
    *   **云端模式 (Cloud)**：使用 OpenAI Whisper API，响应速度快，适合网络环境良好且对精度有一定要求的场景。基于时长计费。
    *   **本地模式 (Local)**：利用本地算力（支持 MLX 加速等），保护隐私且完全免费。

### 🤖 LLM 深度加工 (Semantic Processing)
*   **语义段落分割**：不只是简单的文本堆砌，利用 LLM（如 GPT-4o-mini）将转录文本按逻辑语义分割为自然段落。
*   **文本润色与纠错**：
    *   自动修正同音字、多音字错误。
    *   自动添加符合中文语境的全角标点符号。
    *   在维持原意和字数的基础上，合并破碎短句。
*   **自适应语言强制控制**：
    *   系统会根据视频标题自动识别简/繁体。
    *   除非检测到繁体标题，否则强制通过 LLM 指令确保输出为**简体中文**。

### 📖 沉浸式阅读体验
*   **音影文同步跳转**：点击文本中的任何句子，媒体播放器（视频或音频）会自动跳转到对应的起始时间点。
*   **实时阅读高亮**：播放时，系统根据当前播放进度自动高亮正在阅读的文本段落，并应用平滑的视觉反馈。
*   **多端适配**：响应式 UI 设计，完美支持桌面端和移动端浏览器。

### 📤 导出与协作
*   **多格式导出**：支持一键导出 **SRT** (字幕)、**VTT** (Web 字幕) 和 **TXT** (纯文本) 格式。
*   **快捷剪贴板**：针对非 HTTPS 安全上下文（由于局域网 IP 访问）实现了健壮的复制回退机制，确保“复制全文”功能在任何环境下可用。

---

## 2. 开发者模式 (Developer Mode)

通过在 URL 后添加 `?role=dev` 访问，可开启以下高级特性：
*   **处理消耗统计**：查看每项任务的媒体时长、LLM Token 消耗以及预估的美元成本。
*   **处理历史管理**：直观查看详细的后台处理状态和统计汇总。
*   **系统日志监控**：界面显示后台任务 ID 和实时进度百分比。

## 3. 技术栈
*   **Frontend**: Next.js, Tailwind CSS, Lucide React.
*   **Backend**: FastAPI (Python), Uvicorn.
*   **Processors**: OpenAI API (Whisper/GPT), FFmpeg, yt-dlp.
