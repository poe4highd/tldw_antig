# AI Agent Dev Preferences (AI 协作偏好与规范)

本文档记录了本项目中与 AI Agent 协作时的习惯、目录结构以及推荐的开发流程。

## 1. 目录结构规范

为了保持项目整洁并清晰区分“开发过程记录”与“用户文档”，我们遵循以下目录分配原则：

### 1.1 `dev_docs/` (开发文档文件夹)
*   **用途**：保存所有与**开发过程**直接相关的辅助文档（如计划、记录、演示、测试报告）。
*   **命名习惯**：
    *   **时间戳区分**：同类型交互文件（如多个 Plan）**必须包含 8 位日期时间戳**（格式：`YYYYMMDD`），以便按时间顺序区分不同阶段的任务。
    *   **简短精炼**：文件名应尽量简短，准确描述任务核心内容，避免冗长。
    *   **示例**：`plan_auth_20260117.md`, `walkthrough_ui_20260116.md`。

### 1.2 `docs/` (项目使用文档文件夹)
*   **用途**：保存**面向最终用户或系统管理员**的正式文档。
*   **内容包括**：
    *   部署指南 (Deployment Guides)
    *   功能使用手册 (Feature Guides)
    *   存储管理规范 (Storage Management)
    *   产品路线图 (Roadmap)
*   **要求**：此目录下的文档应保持结构清晰、内容精炼，方便查阅。

### 1.3 `README.md` (项目总入口)
*   作为项目的“门户”，`README.md` 必须包含指向上述两个核心目录的索引记录，确保开发者和用户都能快速找到所需资源。

## 2. Agent 交互习惯

### 2.1 任务驱动流程
*   **任务拆解**：在开始复杂工作前，Agent 应优先更新 `task.md`。
*   **方案确认**：对于涉及重大逻辑变动的任务，Agent 应在 `dev_docs/` 下创建或更新 `implementation_plan.md` 并请求用户复核。

### 2.2 开发记录
*   **执行与验证**：在完成功能开发后，Agent 应通过 `walkthrough.md` 记录验证结果（包含录屏、截图或日志片段），并存档于 `dev_docs/`。

## 3. 技术实践偏好 (由历史经验总结)

*   **前后端分离**：前端 Vercel + 后端内网穿透（Cloudflare Tunnel）。
*   **环境隔离**：敏感信息仅存本地 `.env`。
*   **防止 Hydration 错误**：避免在服务端生成随机数或不确定的 DOM 属性。
*   **自适应处理**：LLM 任务应具备语言自动识别与处理能力，保持转录语言的一致性。
